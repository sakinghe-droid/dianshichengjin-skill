# 点石成金 · 已知局限

## DDX默认值陷阱

当无外部DDX数据源时，引擎填充默认值 `ddx_5d=0.3, ddx_10d=0.1`。此值**不代表真实DDX**——真实DDX可能好得多也可能差得多。

**实盘案例（常友科技301557, 2026-06-26）**：
- 引擎输出: DDX 5日0.3 / 10日0.1 → 判"中性偏弱"
- 扣子抓取真实DDX: 10日1.45→5日0.67→3日0.085 → 逐级缩水，主力减仓

默认值0.3/0.1比真实3日0.085**高估了3倍**，导致引擎漏判主力撤退信号。

## E5低位≠可以买

E5是**位置维度**（安全边际），不是**质量维度**。以下情况E5低位仍应否决：
1. DDX逐级缩水（主力出货）
2. 基本面恶化（Q1亏损/现金流转负）
3. 非主线/退潮期（D象限或板块退潮）
4. 资金拆解显示机构净流出

**常友科技案例**: E5=22.6%低位 + DDX逐级缩水 + Q1亏损 + 风电非主线 → 三重否决。

## 缺失维度

| 缺失 | 影响 | 检测方法 |
|------|------|---------|
| DDX多周期趋势 | 无法识别逐级缩水/递增 | 扣子mx-data或手工连查3-5天 |
| 资金四层拆解 | 无法区分机构vs游资 | 同花顺L2/扣子 |
| PE/现金流/Q1 | 无法排除基本面恶化 | 扣子/东财F10 |
| 概念纯度 | 无法排除蹭概念 | 扣子/年报营收占比 |
